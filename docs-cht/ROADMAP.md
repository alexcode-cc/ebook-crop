# 功能藍圖

本文件規劃 ebook-crop 未來的功能方向與改進建議，作為後續開發的參考依據。

---

## 階段一：品質與測試基礎

在新增功能前，先建立自動化測試以確保穩定性。

| 項目 | 說明 | 優先度 |
|------|------|--------|
| ✅ pytest 框架 | 建立 pytest 測試架構，含測試 fixtures 與範例 PDF | 高 |
| ✅ Config 單元測試 | 測試 `parse_rotation_list`、`format_rotation_display`、`load_config` | 高 |
| ✅ Rotation 單元測試 | 測試 `_get_rotated_page_rect`、寬高對調邏輯 | 高 |
| ✅ Crop 單元測試 | 測試 `_apply_crop` 邊界條件（空頁面、零留白） | 高 |
| ✅ 整合測試 | 端對端：載入設定、裁切 PDF、驗證輸出 | 中 |
| ✅ 邊界測試 | 單頁、空檔案、加密 PDF、超大檔案 | 中 |
| ✅ CI 測試流程 | 將 pytest 加入 CI 工作流程，於 Python 3.10/3.11/3.12 執行 | 高 |
| ✅ 程式碼覆蓋率 | 在 CI 中加入覆蓋率報告 | 低 |

## 階段二：使用者體驗改善

提升 CLI 操作便利性與回饋。

| 項目 | 說明 | 優先度 |
|------|------|--------|
| ✅ `--version` 旗標 | 顯示目前版本（`ebook_crop.__version__`） | 高 |
| ✅ 進度條 | 批次處理時顯示進度（如 `rich` 或 `tqdm`） | 中 |
| ✅ 詳細/靜默模式 | `-v` 顯示詳細日誌、`-q` 靜默模式 | 中 |
| ✅ 預覽模式 | `--dry-run` 預覽設定與影響頁面，不實際處理 | 中 |
| ✅ 留白單位支援 | 設定檔中可使用 `cm`、`mm`、`inch`（自動轉換為點） | 中 |
| ✅ 設定檔驗證 | 載入時驗證設定值，對無效留白、頁數範圍顯示明確錯誤訊息 | 中 |
| ✅ 彩色輸出 | 終端機輸出以顏色區分警告與錯誤 | 低 |

## 階段三：核心功能擴充

擴展裁切與旋轉功能。

| 項目 | 說明 | 優先度 |
|------|------|--------|
| 自動偵測留白 | 透過 PyMuPDF 文字/繪圖擷取分析頁面內容邊界，建議裁切量 | 高 |
| 每頁不同留白 | 擴充 `[[margins]]` 設定，支援每頁或每範圍的留白覆寫 | 中 |
| 奇偶頁留白 | 奇數頁與偶數頁使用不同留白（適用書籍式掃描） | 中 |
| 裁切預覽 | 將單頁渲染為圖片（PNG），顯示裁切/旋轉結果後再進行完整處理 | 中 |
| tomllib 支援 | Python 3.11+ 使用 stdlib `tomllib`，3.10 保留 `tomli` 作為後備 | 低 |
| 可設定 garbage | 將 PyMuPDF `garbage` 等級（0-4）開放為設定選項，優化輸出 | 低 |
| 中繼資料保留 | 處理時保留或更新 PDF 中繼資料（標題、作者、主題） | 低 |

## 設計原則

實作新功能時，遵循以下原則：

1. **向後相容**：新增設定選項必須為可選；現有 `config.toml` 無需修改即可繼續使用
2. **最小依賴**：優先使用標準函式庫，新增依賴需有充分理由
3. **優雅失敗**：無效設定或不支援的 PDF 功能應產生明確錯誤訊息，而非程式崩潰
4. **雙語維護**：所有使用者介面字串、文件與 commit 訊息均支援英文與繁體中文
